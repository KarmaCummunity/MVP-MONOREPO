## 🚨 Critical Vulnerabilities
 * Committed database backup artifacts expose user PII and credential material. `apps/api/data-backups/20251224-162421/users.json` includes real-looking emails, phone values, notifications, and an argon2 `passwordHash` (lines 3-31); the same backup tree includes user/chat/notification/CRM exports. File: `apps/api/data-backups/20251224-162421/*` -> Action: remove committed backups, add backup/export paths to `.gitignore`, rotate or invalidate affected credentials/sessions, and purge repository history if this repository is public or broadly shared.
 * A development JWT signing secret is committed in deploy configuration. File: `apps/api/railway.toml.dev` lines 33-34 -> Action: rotate the secret, replace the committed value with a placeholder, and require Railway/environment secrets for all deployable environments.
 * Root-admin identity is hardcoded and auto-promoted during normal auth flows, conflicting with SRS SEC-003.1 ("Admin access is role-based only — no hardcoded email checks"). Files: `apps/api/src/config/kc-root-admin.defaults.ts` line 6; `apps/api/src/services/user-auth.service.ts` lines 101-102 and 212-220 -> Action: remove email-based promotion from register/login, bootstrap only from `ROOT_ADMIN_EMAIL`, and fail closed when it is unset in production.
 * Dependency audit is currently failing. `npm audit --workspaces --audit-level=high --json` reports 67 total vulnerabilities including 1 critical `handlebars` advisory in the full dependency tree; `npm audit --workspaces --omit=dev --audit-level=high --json` reports 42 production-scope advisories with 33 high findings, notably the React Native/Expo chain and `fast-xml-parser` entity-expansion advisories. File: `package-lock.json` -> Action: restore a security-audit gate and update/override affected dependency chains with Expo/React Native compatibility validation.

## 🐞 Bugs & Logic Issues
 * Admin task infinite scroll can skip pages under concurrent `onEndReached` calls. `loadMore` only checks `loading`, but load-more requests never set `loading`; overlapping page increments race through `fetchTasksSeqRef`, and older page responses can be dropped as stale. File: `apps/mobile/screens/admin/useAdminTasksScreen.tsx` lines 220-225 and 352-408 -> Recommendation: add a dedicated `loadingMore` flag/ref and do not increment `page` until the active load-more request resolves.
 * Admin task refresh can desynchronize pagination after loading page 2+. A full refresh fetches offset 0 and replaces rows while retaining the old `page`; the next load-more can jump to the wrong offset. File: `apps/mobile/screens/admin/useAdminTasksScreen.tsx` lines 368-395 and 479-483 -> Recommendation: centralize "replace list" behavior so every full refresh resets `page` to 0 and `hasMore` consistently.
 * Editing a post image can persist device-local `file://`/`content://` URIs and make images unavailable to other clients; the API also replaces the full `posts.images` array with at most one URL, dropping additional images. Files: `apps/mobile/components/Feed/EditPostModal.tsx` lines 73-75 and 117-119; `apps/api/src/controllers/posts.controller.ts` lines 2349-2356 -> Recommendation: upload picked assets through the normal storage pipeline before save, validate remote `http(s)` URLs, and preserve/accept arrays if multi-image posts are supported.
 * Trump ride search mode exposes an `includePast` filter that has no effect. Search mode returns after fetching posts and `applyPostModeFilters` only applies request/no-cost filters. Files: `apps/mobile/donationScreens/trump/useTrumpScreenData.ts` lines 36-51 and 105-107; `apps/mobile/donationScreens/trump/getFilteredRidesForTrumpScreen.ts` lines 33-41 -> Recommendation: either hide `includePast` in search mode or thread a date-based filter through the post-query/client-filter path.
 * `PUT /posts/:id` still treats `body.user_id` as an identity fallback after `JwtAuthGuard`. Current guard behavior likely supplies `req.user`, but a future guard/payload regression could make ownership checks depend on spoofable request body data. File: `apps/api/src/controllers/posts.controller.ts` lines 2300-2305 -> Recommendation: derive identity only from authenticated request context and reject mismatched `body.user_id` if kept for compatibility.
