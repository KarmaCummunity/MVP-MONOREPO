# 🛡️ Daily Audit Report: May 6, 2026

## 🚨 Critical Vulnerabilities
* Unauthenticated chat routes can disclose and mutate private conversations because `ChatController` uses `OptionalAuthGuard` and several guest branches trust route/body identifiers. Concrete scenarios: `GET /api/chat/conversations/user/:userId` returns another user's conversation list without a token, `GET /api/chat/conversations/:conversationId/messages` returns messages for any known conversation UUID, `POST /api/chat/messages` can send as an arbitrary `sender_id`, and read-receipt endpoints accept `body.user_id` as identity. - File: `apps/api/src/controllers/chat.controller.ts:45-47`, `apps/api/src/controllers/chat.controller.ts:246-338`, `apps/api/src/controllers/chat.controller.ts:341-438`, `apps/api/src/controllers/chat.controller.ts:441-668`, `apps/api/src/controllers/chat.controller.ts:671-763` -> Action: Require strict JWT authentication for chat reads/writes, remove unauthenticated `body.user_id` identity fallbacks, and enforce participant checks for every conversation/message operation.
* Public posts reads can trigger destructive schema migration. `GET /api/posts` calls `ensurePostsTable()`, and the migration helper drops `posts CASCADE` when expected columns are missing, so a normal request against a legacy/mismatched schema can cause irreversible data loss. - File: `apps/api/src/controllers/posts.controller.ts:67-79`, `apps/api/src/controllers/posts.controller.ts:830-843` -> Action: Remove destructive DDL from request paths; move schema changes to controlled migrations/startup checks that fail closed instead of dropping tables.
* Web refresh/access tokens are persisted in AsyncStorage, exposing long-lived sessions to any XSS, compromised browser extension, or same-origin script compromise in the web build. - File: `apps/mobile/auth/services/tokenManager.web.ts:1-26` -> Action: Move web session handling to httpOnly/SameSite secure cookies or another storage design that keeps refresh tokens unavailable to JavaScript.
* CI SonarCloud workflows pin `SonarSource/sonarcloud-github-action` to mutable `@master`, creating a supply-chain risk where an upstream branch change can run arbitrary code with CI token/secrets access. - File: `apps/mobile/.github/workflows/sonar.yml:24-30`, `apps/api/.github/workflows/sonar.yml:32-36` -> Action: Pin to a stable release tag or full commit SHA and review update cadence.

## 🐞 Bugs & Logic Issues
* Navigation guards fail open if a guard throws, so an unexpected guard error can allow protected navigation instead of blocking it. - Recommendation: Treat guard exceptions as denied navigation with a safe redirect, and test admin/profile guard failure behavior. File: `apps/mobile/utils/navigationGuards.ts:237-260`.
* Admin organization approvals diverge from the global admin definition and loads data before denying access. `super_admin`, `org_admin`, or root-email admins can be rejected because the screen only checks `roles.includes('admin')`, while non-admin visits still trigger `db.listOrgApplications('admin_org_queue')` during mount. - Recommendation: Reuse the shared `isAdmin` derivation and gate `load()` before calling admin data APIs. File: `apps/mobile/screens/admin/AdminOrgApprovalsScreen.tsx:35-52`, `apps/mobile/screens/admin/AdminOrgApprovalsScreen.tsx:108-114`, `apps/mobile/stores/userStore.ts:824-835`.
* Post detail gallery image URLs are rendered directly from API-provided `item.images` / `item.thumbnail` without the URL validation used elsewhere for avatars. - Recommendation: Filter gallery URIs through the existing safe-image URL validator before passing them to `Image`. File: `apps/mobile/screens/postDetail/PostDetailBody.tsx:74-77`, `apps/mobile/screens/postDetail/PostDetailBody.tsx:426-431`.
* Several backend catch blocks return raw provider/database error messages to clients, disclosing backend details even when stack traces are only logged. - Recommendation: Return stable client-safe error codes/messages and keep raw details in server logs. File: `apps/api/src/controllers/chat.controller.ts:331-337`, `apps/api/src/controllers/chat.controller.ts:431-437`, `apps/api/src/controllers/chat.controller.ts:653-667`, `apps/api/src/controllers/posts.controller.ts:1167-1182`.

## 🏗️ Architecture & Clean Code
* HTTP controllers are coupled to database schema management. The posts controller performs DDL and destructive migration checks from read endpoints, violating the separation between request handling and infrastructure migration operations. File: `apps/api/src/controllers/posts.controller.ts:63-94`, `apps/api/src/controllers/posts.controller.ts:830-843`.
* Chat authorization is implemented through repeated controller-level `if (authenticatedUserId) ... else ...` branches under `OptionalAuthGuard`, which has already produced confidentiality/integrity gaps. File: `apps/api/src/controllers/chat.controller.ts:45-47`, `apps/api/src/controllers/chat.controller.ts:246-763`.
* Admin screens are registered in the same authenticated/guest stack, with deep-link mappings for admin routes and per-screen guard reliance. This increases blast radius when any screen guard is incomplete or fails open. File: `apps/mobile/navigations/MainNavigator.tsx:105-155`, `apps/mobile/utils/linkingConfig.ts:217-219`, `apps/mobile/hooks/useAdminProtection.ts:58-65`.
* Large legacy controllers/services remain outside the target clean-architecture layering and size guidance, especially `posts.controller.ts` and the mobile `databaseService` monolith. Continue reducing these through scoped refactors rather than opportunistic rewrites.

## 🧹 Technical Debt & Maintenance
* Local Docker Compose exposes Postgres and Redis ports with predictable development credentials/no Redis password. This is acceptable only for isolated local development; accidental exposure on a shared host gives attackers known access paths. File: `apps/api/docker-compose.yml:6-22`.
* One-off admin scripts contain hardcoded real email identities and can mutate privilege hierarchy if run against the wrong database. File: `apps/api/update-karmacommunity-admin.ts:10-12`, `apps/api/src/scripts/fix-super-admins.ts:29-53`.
* `reset-all-local-data.sh` loads `.env` values via `export $(grep ... | xargs)`, which is brittle for quoted values and unsafe if malformed/untrusted `.env` content is present. File: `apps/api/scripts/reset-all-local-data.sh:21-29`.
* A committed ESLint report leaks developer workstation paths (`/Users/navesarussi/...`) and should not live in source control. File: `apps/api/eslint-report.json:5`.
* Existing tech-debt log items for `apps/mobile/utils/databaseService.ts`, orphan donation screen wrappers, and admin dynamic table APIs remain relevant and were not expanded in this audit.
